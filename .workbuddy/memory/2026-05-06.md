## KeePassHO 附件大小展示功能

在 EntryView 页面的附件条目中增加了附件大小显示，修改了以下文件：

1. **StringUtils.ets** — 新增 `formatFileSize(size: number): string` 通用方法，将字节数格式化为 B/KB/MB/GB
2. **EntryViewFieldItem.ets** — FieldView 类新增 `fileSizeStr: string` 属性；build() 中值区域渲染逻辑改为附件类型优先显示 `fileSizeStr`
3. **EntryView.ets** — `addBinary()` 方法中计算 binaryFile 的 byteLength 并通过 `StringUtils.formatFileSize()` 格式化后赋值给 `fieldView.fileSizeStr`

渲染效果：附件条目值区域显示格式化后的文件大小（如 "1.5 KB"），替代原来的空字符串。

## 附件图片预览弹窗功能

新增图片格式附件的预览功能，修改/新增以下文件：

1. **FilenameUtils.ets** — 新增 `isImage(fileName): boolean` 静态方法，支持 jpg/jpeg/png/gif/bmp/webp/ico 格式判断
2. **ImageViewerDialog.ets**（新建）— 图片预览弹窗组件：
   - @CustomDialog + DialogAlignment.Bottom 从底部弹出
   - ArrayBuffer → image.createImageSource → PixelMap 渲染
   - PinchGesture 双指缩放（1.0x~5.0x）、TapGesture 双击还原/放大
   - PanGesture 单指拖拽（缩放 >1.0 时）
   - 底部显示文件大小 + 保存按钮（复用 FileUtils.saveLocalFile）
   - 黑色背景、右上角关闭按钮
3. **EntryViewFieldItem.ets** — FieldView 新增 `isImageFile` 属性；图片附件点击整行打开预览弹窗
4. 图片附件图标统一使用 `ic_color_paperclip`（与普通附件一致，属于 `ic_color_` 预着色系列图标）
5. 图片附件保留右侧下载按钮

## 图片预览弹窗修复（第二轮）

修复图片附件预览的5个问题：

1. **图标样式不一致** → 统一使用 `ic_color_paperclip`，移除 `file_image` 图标（`ic_color_*` 为预着色图标，不需要 fillColor）
2. **图片附件缺失下载按钮** → 恢复图片附件的下载按钮，与普通附件一致
3. **弹窗样式不一致** → 从黑色全屏改为项目统一弹窗风格：`bg_primary` 背景、`borderRadius: 16`、`ShadowStyle.OUTER_DEFAULT_MD`、高度 80%
4. **附属功能冗余** → 移除文件名、文件大小、下载按钮，仅保留预览和缩放功能
5. **PixelMap 创建错误** → 最终方案：ArrayBuffer → Base64 Data URI → `Image(this.imageDataUri)`，绕过 ImageSource/PixelMap 整个链路。在 OpenHarmony 上 `image.createImageSource` 解码某些图片会失败，而 `data:image/...;base64,...` 方式兼容性最好
6. `util.Base64Helper` 需要 `new util.Base64Helper()` 实例化，不是静态类；`encodeToStringSync(src: Uint8Array): string`

关键经验：
- 项目中 `ic_color_*` 前缀图标是预着色资源，不需要 `.fillColor()`；`ic_*` 单色图标才需要 `.fillColor($r('app.color.text_primary'))`
- `image.createImageSource(ArrayBuffer)` 在某些场景下可能失败，使用临时文件+fd方式更可靠（与 ImagePickerUtils 一致）
- 项目弹窗统一样式：`bg_primary` 背景、16圆角、OUTER_DEFAULT_MD阴影、56高标题栏、24/16 padding
